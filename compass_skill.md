# 新品/老品设计需求输出（FABE文案方法）

基于 FABE（Features-Advantages-Benefits-Evidence）方法论，从亚马逊商品信息、用户评价和社媒讨论中提取洞察，
按固定模板输出产品设计需求文档。

## 工作流程

### 第一步：信息收集

从用户获取以下信息，信息不完整时主动询问：

**必填信息：**
- 产品SKU
- 产品成本与售价
- 产品参数（品牌、功能、材质、尺寸、重量、防水、噪音、充电、续航等）
- 目标上架站点（如 AC-US, BV-US, BV-UK, 欧洲）

**可选但强烈建议提供：**
- 产品图片（用于AI视觉分析，帮助理解产品外观和功能）
- 产品核心卖点方向（如有预设）
- 竞品链接或竞品信息

**内置信息源（技能会自动使用）：**
- 亚马逊产品Listing信息（产品描述、规格、图片、A+内容）
- 用户评论数据（好评提炼优势、差评挖掘痛点）
- 社媒讨论（Reddit、论坛等关于该品类/产品的讨论）
- 竞品对比数据

当用户提供的产品信息不完整时，先尝试通过内置信息源（WebSearch/WebFetch）补充，
无法获取的再向用户追问。

### 第二步：用户评价与痛点分析

1. 收集目标产品的亚马逊评论（好评和差评），按维度归类
2. 对好评反向推导——为什么用户觉得好？提炼出产品优势
3. 对差评正向推导——用户遇到什么问题？提炼出核心痛点
4. 结合社媒讨论，补充用户未在评论中明确表达的需求

分析输出格式参见 [fabe-template.md](references/fabe-template.md) 中的「用户真实反馈」和「核心用户痛点」部分。

### 第三步：人群与场景推导

基于产品参数和用户评价分析，推导：
1. **目标用户群体** —— 谁在什么状态下使用这个产品？
2. **核心使用场景** —— 用户在什么具体时刻使用它？
3. **产品定位** —— 用第一人称写一句话，说明产品为谁解决什么问题

核心推导链路：
```
产品基础信息分析 → 总结产品特点 → 目标人群&场景化痛点匹配/挖掘
→ 回答"在哪个具体时刻，解决了用户什么具体问题？"
→ 展示场景化解决方案（定价+内容（关键词+场景））
```

完整推导框架：
```
产品是什么 → 谁会买 → 为什么现在会买 → 产品解决什么问题
→ 怎么解决 → 凭什么解决 → 用户怎么使用 → 用户怎么搜索
→ 如何提升完整体验 → 别人怎么评价
```

### 第四步：FABE卖点推导

按照 **1个核心卖点 + 2个辅助卖点** 的结构：

**F（Features 特征）：** 产品本身有什么？从参数中提取客观事实。
**A（Advantages 优势）：** 这些特征比竞品好在哪？差异化点是什么？
**B（Benefits 利益）：** 给用户带来什么好处？解决什么具体问题？
**E（Evidence 证据）：** 用什么证明？用户评论、数据对比、测试结果。

每个卖点需要输出：
1. 解决方案（如何解决用户问题）
2. 卖点表达文案（中文+英文）

### 第五步：输出完整FABE模板

按 [fabe-template.md](references/fabe-template.md) 格式输出完整FABE分析，
包含：产品基础信息、用户群体、用户反馈/痛点、产品定位、解决方案、场景表达、关键词、多语言详情页文案、评论设计方向。

### 第六步（可选）：输出设计需求文档

如用户需要图片/详情页设计方向，按 [design-requirement-template.md](references/design-requirement-template.md) 格式输出，
为每张主图和详情页指定：尺寸、是否动图、是否需要文案、卖点类型、痛点映射、需求说明、多语言文案。

## 输出语言

所有文案默认输出中英双语。如用户指定目标市场（欧洲站等），追加对应语言：
- 德语
- 法语
- 荷兰语
- 西班牙语
- 意大利语

## 输出方式

### 钉钉在线表格（推荐）

使用 `dws sheet range update` 写入钉钉在线表格，与原始模板一致的多 sheet 表格：
- Sheet 1 = FABE 完整分析
- Sheet 2 = 设计需求文档

> **重要已知限制**：`dws sheet csv-put` 的 `--csv -`（stdin模式）和 `--csv @filepath` 在当前版本均不工作，
> 会将 `-` 或 `@path` 当作字面文本写入。**必须使用 `dws sheet range update` + `--values` JSON数组逐段写入。**
> 建议使用 Python subprocess 分批调用，每批 10-30 行。

创建流程：
1. `dws sheet create --name "FABE设计需求-{产品名}"` 创建文档
2. `dws sheet update --sheet-id ... --title "FABE"` 重命名默认sheet
3. `dws sheet new --node ... --name "设计需求文档"` 创建第二sheet
4. `dws sheet range update --range "A1:B23" --values '[[...]]'` 逐段写入数据

### Markdown 文件（备选）

将完整内容写入 `.md` 文件，适合离线查看或存档。
