# 极端/领先用户查找-分析技能

## 技能定位

本技能基于**Eric von Hippel的领先用户理论**，提供系统化的极端用户识别与洞察提取方法。从VOC数据中找到那些"需求领先于主流市场"且"有强烈动机自己解决问题"的用户，提取他们的深层需求和创新机会。

## 理论基础：von Hippel 领先用户理论

### 核心观点
- 创新不一定来自制造商的R&D，而是来自**领先用户**
- 领先用户的需求揭示了**未来1-2年后主流市场**的需求
- 领先用户会**自己发明解决方案**（workarounds）

### 两大识别标准
1. **时间领先性**：用户的需求领先于主流市场
2. **高收益预期**：如果用户自己开发解决方案，会获得高收益

### 经典案例
- 山地自行车：加州自行车爱好者自己改装 → 厂商商业化
- 3M便利贴：实验室研究员自己制作临时标记 → 3M商业化
- 医疗器材：医生自己修改手术工具 → 厂商改进

## 核心能力

1. **三层渐进筛查**：从粗筛到精筛，系统化识别极端用户
2. **理论-数据匹配**：将von Hippel理论标准映射到具体数据源特征
3. **信号检测**：自动识别workarounds、未来导向需求、极端情感等信号
4. **洞察提取**：从极端用户表达中提取产品创新机会
5. **可视化报告**：生成符合Consulting风格的HTML报告

## 使用流程

### 步骤1：理解用户输入

用户可能以以下方式提问：
- "帮我找一下飞机杯的极端用户"
- "从Reddit评论中识别领先用户"
- "哪些用户的需求是未来的主流需求？"
- "帮我分析一下用户的workarounds"

**关键信息提取**：
- 产品类型/领域
- 数据源类型（Reddit/Amazon/访谈/社媒）
- 分析深度（快速筛查/深度洞察）

### 步骤2：选择匹配方法

根据数据源类型，选择不同的匹配方法：

#### 方法1：标签模块筛查（粗筛）
**适用**：已用9大标签模块打标的数据

| 模块 | 极端标准 | 领先用户信号强度 |
|------|---------|-------------------|
| ② 购买动机 | 提到3个以上动机 | ⭐⭐ （高动机 = 高收益预期） |
| ⑤ 使用阻碍 | 提到3个以上阻碍 | ⭐⭐⭐ （阻碍多 = 高收益预期） |
| ⑥ 克服方式 | **描述创意workaround** | ⭐⭐⭐⭐⭐ （最强信号！） |
| ⑦ 用户忠诚度 | 极端忠诚 OR 极端不满 | ⭐⭐⭐ （极端 = 高情感投入） |
| ⑧ 产品改进建议 | 建议非常详细/具体 | ⭐⭐⭐⭐ （具体 = 用户认真思考过） |

**逻辑**：如果用户在9大模块中，至少有2个模块表现出"极端"特征 → 进入第二层筛查

#### 方法2：文本信号检测（精筛）
**适用**：原始VOC文本数据

**信号1：Workaround描述（最强信号）**
```
匹配模式：
"我自己..." / "我用了XX代替..." / "我改装了..." / "我发现了一种方法..." 
"I ended up using..." / "My workaround is..."

理论对应：高收益预期 → 用户自己动手解决了
```

**信号2：未来导向需求**
```
匹配模式：
"我希望..." / "为什么没有..." / "如果有人能..." / "未来..." 
"I wish..." / "Why doesn't anyone..."

理论对应：时间领先性 → 用户在描述未来需求
```

**信号3：极端情感强度**
```
匹配模式：
极度正面：改变生活 / 强烈推荐 / 完美 / game changer
极度负面：最差 / 浪费钱 / 设计缺陷 / 不会再买

理论对应：高收益预期 → 情感投入深
```

**信号4：社交影响力**
```
匹配模式：
"推荐给X个人" / "所有人都应该..." / "我分享了..."

理论对应：领先用户会主动扩散创新
```

### 步骤3：领先用户分类（von Hippel 2×2矩阵）

```
           高时间领先性
               ↑
               │
高收益预期 ←───┼───→ 低收益预期
               │
               ↓
           低时间领先性

分类标准：
├── 高时间领先 + 高收益 → ✅ 真正的领先用户（优先级最高）
├── 高时间领先 + 低收益 → ⚠️ 趋势观察者（关注未来，但无强烈动机）
├── 低时间领先 + 高收益 → ⚠️ 重度用户（需求当前，但动机强）
└── 低时间领先 + 低收益 → ❌ 普通用户（忽略）
```

### 步骤4：提取创新洞察

从领先用户的表达中提取：

#### 洞察1：Workarounds → 产品功能机会
```
用户原话："我自己加了一个加热套"
↓
洞察：用户对温度有强需求，但产品未满足
↓
创新机会：内置加热功能
```

#### 洞察2：未来需求表达 → 趋势预测
```
用户原话："我希望它能自动调节紧度"
↓
洞察：未来主流需求是"智能化+个性化"
↓
趋势预测：1-2年后，智能调节将成为标配
```

#### 洞察3：极端忠诚 → 核心价值主张
```
用户原话："这产品改变了我的生活，强烈推荐给所有人"
↓
洞察：用户真正在意的是"真实感+隐私保护"
↓
核心价值：真实感 + Discreet设计
```

#### 洞察4：极端批评 → 设计缺陷暴露
```
用户原话："清洗太麻烦，每次都要洗半天，设计有问题"
↓
洞察：清洗流程是核心痛点
↓
设计改进：自清洁功能 / 可拆卸清洗设计
```

### 步骤5：生成可视化HTML报告

使用 `assets/report_template.html` 作为模板，生成包含以下内容的报告：

1. **Hero区域**：
   - 标题："极端/领先用户分析报告 - [产品名称]"
   - 副标题：数据分析范围、用户总数、领先用户数
   - 核心数字：领先用户占比、高价值洞察数

2. **理论基础卡片**：
   - von Hippel领先用户理论简介
   - 两大识别标准说明

3. **三层筛查流程展示**：
   - 第一层：标签模块筛查结果
   - 第二层：文本信号检测结果
   - 第三层：领先用户分类矩阵

4. **领先用户画像**（核心内容）：
   - 每个领先用户的完整画像
   - 包含原始语录引用
   - 标注信号类型和强度

5. **创新洞察提取**：
   - Workarounds → 产品功能机会
   - 未来需求 → 趋势预测
   - 极端忠诚 → 核心价值主张
   - 极端批评 → 设计缺陷暴露

6. **行动建议**：
   - 短期可实施方案（1-3个月）
   - 中期研发方向（3-6个月）
   - 长期趋势布局（6-12个月）

## 数据源适配指南

| 数据源 | 时间领先性检测 | 高收益预期检测 | Workaround检测 | 推荐用法 |
|--------|----------------|----------------|----------------|-----------|
| **Reddit** | ✅ 有时间戳，可分析趋势 | ✅ 情感强度高 | ✅✅ 用户会详细分享DIY | **首选数据源** |
| **Amazon评价** | ❌ 无时间戳 | ✅ 极端评分+详细文字 | ⚠️ 偶尔有 | 次要数据源 |
| **访谈记录** | ❌ | ✅✅ 深度挖掘 | ✅✅ 如果专门问了 | 补充数据源 |
| **社媒反馈** | ⚠️ 取决于平台 | ✅ | ⚠️ 取决于平台 | 补充数据源 |

## 输出质量标准

**好的领先用户识别报告特征**：
- ✅ 理论支撑（基于von Hippel理论）
- ✅ 信号明确（每个领先用户都有明确信号证据）
- ✅ 洞察可执行（不是"用户有需求"，而是"具体什么需求+如何满足"）
- ✅ 趋势可预测（基于领先用户需求，预测未来1-2年主流趋势）

**避免的输出**：
- ❌ 只有理论没有数据支撑
- ❌ 把所有极端用户都归为"领先用户"（需要用2×2矩阵分类）
- ❌ 洞察过于抽象（"用户希望更好"→ 应该具体化为"用户希望加热到38°C"）
