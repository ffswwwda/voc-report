# VOC分析协议 · 营销建议目的

> 本文件为AI分析执行协议。AI读取本协议 + 原始VOC数据，直接输出成品分析报告。

---

## 元数据

| 字段 | 内容 |
|------|------|
| 分析目的 | 营销建议 |
| 核心问题 | 用户最在意什么价值？如何打动目标用户？ |
| 适用场景 | 文案撰写、受众定位、卖点提炼、广告投放 |
| 预计分析时长 | 30-45分钟 |

---

## 输入数据规范

### 主数据源
- **Amazon评价（4-5星）**：用户喜欢什么、为什么推荐、使用场景描述
- **社媒分享帖**：用户主动分享的使用体验、晒图文案

### 辅助数据源
- **Reddit正面讨论**：用户向他人推荐时的表达逻辑
- **Reddit负面讨论**：用户不满的点（反向提炼：避免什么）

### 数据预处理要求
1. **重点保留4-5星评论**（正面价值表达）
2. 保留含"推荐"、"喜欢"、"最好"等关键词的评论
3. 标记评论中的使用场景描述（营销素材来源）

---

## 分析步骤（严格按顺序执行）

### Step 1：标签打标
对每条原始评论，按9大标签模块打标：

| 优先级 | 标签模块 | 打标重点 |
|--------|-----------|-----------|
| ⭐⭐⭐ | ②购买动机 | **核心**：提炼用户购买的核心驱动力（营销卖点来源） |
| ⭐⭐⭐ | ①使用场景 | 提取具体使用场景（广告素材来源） |
| ⭐⭐⭐ | ⑨用户画像/人群 | 识别核心受众特征（受众定位） |
| ⭐⭐ | ③预期符合度 | 找出超出预期的点（差异化卖点） |
| ⭐⭐ | ⑦用户忠诚度 | 识别品牌倡导者（KOC来源） |
| ⭐ | 其余模块 | 辅助上下文理解 |

- **输出**：打标后的结构化数据
- **质量检查**：购买动机是否覆盖了所有评论的80%以上？

### Step 2：JTBD框架分析（营销视角）
- 从正面评论中提取：**用户购买时想要完成的"任务"是什么**
- 重点分析**情感性任务**和**社会性任务**（营销核心）
  - 情感性任务 → 情感营销文案
  - 社会性任务 → 社交传播点
- **输出**：JTBD营销洞察表（任务 → 对应卖点 → 文案建议）

### Step 3：十三种需求分析（价值提炼）
- 将正面评论映射到十三种需求模型
- **重点**：找出用户评价中提及最多的需求（核心卖点）
- **输出**：需求价值矩阵（需求 → 提及频率 → 对应文案建议）

### Step 4：研究员职责视角分析
- 用四象限框架分析用户需求类型
- **重点**：区分"表达出的需求"和"真实需求"
- **输出**：需求层次洞察（指导营销策略）

### Step 5：整合输出成品报告
按以下结构输出（详见「输出报告结构」章节）

## AI执行注意事项

1. **正面数据为主**：本分析以4-5星评论为主，1-3星仅作反向参考
2. **原文引用是核心**：每条卖点提炼必须有用户原文支撑
3. **场景化表达**：使用场景描述直接转化为广告素材建议
4. **区分功能和情感**：功能卖点 vs 情感卖点，分别应用于不同渠道
5. **KOC识别**：高忠诚度用户的评论可直接用作营销素材（需授权）
