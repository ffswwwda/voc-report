# VOC分析协议 · 产品创新目的

> 本文件为AI分析执行协议。AI读取本协议 + 原始VOC数据，直接输出成品分析报告。

---

## 元数据

| 字段 | 内容 |
|------|------|
| 分析目的 | 产品创新 |
| 核心问题 | 未来1-2年的机会在哪？如何差异化突破？ |
| 适用场景 | 差异化策略、竞品突破、新品立项、技术研发方向 |
| 预计分析时长 | 45-90分钟 |

---

## 输入数据规范

### 主数据源
- **Reddit讨论（全量）**：用户深度痛点、DIY方案、未来需求表达
- **Amazon评价（1-3星）**：产品设计缺陷、未满足需求

### 辅助数据源
- **社媒边缘场景讨论**：包容性设计机会
- **竞品评价**：竞品的创新点和缺陷

### 数据预处理要求
1. **重点保留含"wish"、"hope"、"future"、"如果"等未来导向词汇的评论**
2. **重点保留描述workarounds的评论**（创新机会金矿）
3. 保留极端情感强度的评论（极度正面和极度负面）

---

## 分析步骤（严格按顺序执行）

### Step 1：标签打标
对每条原始评论，按9大标签模块打标：

| 优先级 | 标签模块 | 打标重点 |
|--------|-----------|-----------|
| ⭐⭐⭐⭐⭐ | ⑥克服方式 | **最强信号**：用户自己发明的解决方案 → 创新机会 |
| ⭐⭐⭐⭐⭐ | ⑤使用阻碍 | 未满足需求 → 创新方向 |
| ⭐⭐⭐⭐ | ②购买动机 | 理解需求本质 → 创新切入点 |
| ⭐⭐⭐⭐ | ⑦用户忠诚度 | 识别极端用户（领先用户候选） |
| ⭐⭐⭐ | ⑧产品改进建议 | 直接的创新建议 |
| ⭐⭐⭐ | ①使用场景 | 边缘场景 → 包容性创新 |
| ⭐⭐ | 其余模块 | 辅助上下文理解 |

- **输出**：打标后的结构化数据
- **质量检查**：workarounds是否全部提取？

### Step 2：极端用户识别（von Hippel领先用户理论）
- **核心步骤**：用三层筛查法找出领先用户
  1. 标签模块筛查：≥2个模块表现极端
  2. 文本信号检测：**workaround描述**（最强）、未来导向语言、极端情感
  3. von Hippel 2×2矩阵分类
- **重点输出**：
  - 领先用户画像（含原始语录）
  - 提取的workarounds → 产品功能机会
  - 未来需求表达 → 趋势预测（1-2年后主流）
- **理论依据**：von Hippel《领先用户理论》— 领先用户的需求 = 未来主流需求

### Step 3：TRIZ创新方法分析
- 将用户痛点映射到TRIZ 40个发明原理
- **重点原理**（适配成人用品类别）：
  - 原理10：预先作用（预热功能）
  - 原理15：动态性（可调节紧度）
  - 原理17：多维化（多通道设计）
  - 原理23：反馈（压力感应）
  - 原理32：颜色改变（情绪灯效）
- **输出**：基于TRIZ原理的产品创新方案

### Step 4：产品创意罗盘分析
- 应用4象限×20种打法，系统化生成创新方案
- **重点象限**：
  - 痛点驱动（解决现有问题）
  - 场景延伸（拓展使用场景）
  - 体验升级（提升感官体验）
  - 情感连接（建立品牌情感）
- **输出**：创新方案清单（含可行性评估）

### Step 5：十三种需求分析（创新机会映射）
- 将创新方案映射到十三种需求模型
- 评估每个方案的**需求覆盖广度**（能打动多少用户）
- **输出**：创新方案优先级排序

### Step 6：整合输出成品报告
按以下结构输出（详见「输出报告结构」章节）

## AI执行注意事项

1. **workarounds是金矿**：用户自己发明的解决方案，直接对应产品功能机会，必须全部提取
2. **领先用户理论指导**：时间领先性高的用户需求 = 未来1-2年主流需求
3. **TRIZ原理系统化应用**：不要依赖直觉创新，用40个发明原理系统化思考
4. **可行性评估必须客观**：每个创新方案必须评估技术可行性和资源需求
5. **区分渐进式和突破式创新**：渐进式（短期可落地）vs 突破式（中长期研发）

- [ ] 所有workarounds是否全部提取并转化为创新机会？
- [ ] 领先用户分类是否应用了von Hippel 2×2矩阵？
- [ ] TRIZ方案是否对应了具体的发明原理？
- [ ] 创新优先级排序是否有量化依据？
- [ ] 实施路线图是否有时间维度和资源评估？
