# VOC分析协议 · 市场趋势目的

> 本文件为AI分析执行协议。AI读取本协议 + 原始VOC数据，直接输出成品分析报告。

---

## 元数据

| 字段 | 内容 |
|------|------|
| 分析目的 | 市场趋势 |
| 核心问题 | 需求在往哪个方向走？未来1-2年的机会是什么？ |
| 适用场景 | 提前布局、品类扩展、技术研发方向、投资策略 |
| 预计分析时长 | 60-90分钟 |

---

## 输入数据规范

### 主数据源
- **Reddit讨论（全量，有时间戳）**：趋势分析必须有时序数据
- **Amazon评价（近12个月，按月分层）**：需求变化追踪

### 辅助数据源
- **社媒讨论（有时间戳）**：新兴话题追踪
- **Google Trends数据（如有）**：交叉验证趋势

### 数据预处理要求
1. **必须保留时间戳**：趋势分析的核心是时序对比
2. **按月分层**：将评论按月份分组，观察指标变化
3. **重点保留含未来导向词汇的评论**（"wish"、"hope"、"future"、"下一代"）
4. **保留提及新兴竞品/新功能的评论**（趋势信号）

## 分析步骤（严格按顺序执行）

### Step 1：时序数据准备
- 将评论按月份分组（最近12个月）
- 计算每个月的：评论总量、平均星级、关键标签频率
- **输出**：时序基础数据表

### Step 2：标签打标（全量数据）
对每条原始评论，按9大标签模块打标：

| 优先级 | 标签模块 | 打标重点 |
|--------|-----------|-----------|
| ⭐⭐⭐⭐⭐ | ②购买动机 | **趋势核心**：动机变化反映需求变化 |
| ⭐⭐⭐⭐⭐ | ⑧产品改进建议 | **趋势信号**：用户希望新增的功能 = 未来需求 |
| ⭐⭐⭐⭐ | ①使用场景 | 场景扩展 = 品类扩展方向 |
| ⭐⭐⭐⭐ | ⑤使用阻碍 | 新阻碍出现 = 新产品机会 |
| ⭐⭐⭐ | ⑥克服方式 | workaround演变 = 技术方向 |
| ⭐⭐⭐ | ③预期符合度 | 预期变化 = 市场成熟度 |
| ⭐⭐ | 其余模块 | 辅助上下文理解 |

- **输出**：打标后的结构化数据（含时间戳）
- **质量检查**：时序覆盖率（是否有某个月份数据缺失？）

### Step 3：趋势识别分析
- **方法**：对比最近3个月 vs 最远3个月的关键指标
- **关键指标**：
  - 购买动机分布变化
  - 改进建议关键词变化
  - 使用场景新增类型
  - 平均星级变化
- **输出**：趋势识别报告（含图表建议）

### Step 4：极端用户识别（领先用户理论应用）
- **核心**：应用von Hippel领先用户理论
- **时间领先性判断**：领先用户的需求 = 未来1-2年主流需求
- **重点提取**：
  - 领先用户的workarounds → 未来产品功能
  - 领先用户的未来需求表达 → 趋势预测
- **输出**：领先用户趋势洞察

### Step 5：十三种需求分析（需求层次演变）
- 对比不同时间段的需求分布
- 识别需求层次的变化（例如：从"功能"向"情感"演变）
- **输出**：需求层次演变报告

### Step 6：整合输出成品报告
按以下结构输出（详见「输出报告结构」章节）

## AI执行注意事项

1. **时序数据是核心**：没有时间戳的数据无法做趋势分析，必须要求用户提供时序数据
2. **领先用户理论指导**：时间领先性高的用户需求 = 未来主流需求，这是趋势预测的理论基础
3. **区分短期波动和长期趋势**：3个月内的变化可能是波动，6个月以上的变化更可能是趋势
4. **交叉验证**：如有Google Trends数据，用来交叉验证VOC趋势
5. **置信度必须标注**：每条趋势预测必须标注置信度（高/中/低），并说明原因
