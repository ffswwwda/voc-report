# VOC分析协议 · 竞品分析目的

> 本文件为AI分析执行协议。AI读取本协议 + 原始VOC数据，直接输出成品分析报告。

---

## 元数据

| 字段 | 内容 |
|------|------|
| 分析目的 | 竞品分析 |
| 核心问题 | 竞品哪里做得好/差？如何对标和差异化？ |
| 适用场景 | 竞品对标、定价策略、差异化定位、市场进入策略 |
| 预计分析时长 | 45-60分钟 |

---

## 输入数据规范

### 主数据源
- **本产品Amazon评价（全量）**：基线数据
- **竞品Amazon评价（全量）**：对比数据
- **Reddit竞品讨论**：用户对比竞品时的表达

### 辅助数据源
- **社媒竞品对比帖**：用户主动做的竞品对比
- **YouTube/视频评测**：评测类内容的评论区

### 数据预处理要求
1. **必须同时收集本产品 + 至少2-3个主要竞品的评价数据**
2. 标记评论中提及竞品的段落（直接对比证据）
3. 保留用户"比较了A/B/C后选择了X"的决策逻辑描述

---

## 分析步骤（严格按顺序执行）

### Step 1：标签打标（本产品 + 竞品）
对每条原始评论，按9大标签模块打标：

| 优先级 | 标签模块 | 打标重点 |
|--------|-----------|-----------|
| ⭐⭐⭐⭐⭐ | ⑤使用阻碍 | **核心对比维度**：竞品是否解决了这个阻碍？ |
| ⭐⭐⭐⭐⭐ | ③预期符合度 | 超预期的点（差异化优势）vs 不达预期的点（竞品机会） |
| ⭐⭐⭐⭐ | ②购买动机 | 用户选择本产品 vs 竞品的动机差异 |
| ⭐⭐⭐⭐ | ④性价比感受 | 价格敏感度、价值感知对比 |
| ⭐⭐⭐⭐ | ⑧产品改进建议 | 用户希望产品向竞品学习的点 |
| ⭐⭐⭐ | ①使用场景 | 场景覆盖广度对比 |
| ⭐⭐⭐ | ⑦用户忠诚度 | 品牌转换行为分析 |
| ⭐ | ⑥克服方式 | workaround对比 |
| ⭐ | ⑨用户画像 | 不同产品的用户群体差异 |

- **输出**：本产品 + 竞品打标后的结构化数据
- **质量检查**：竞品数据是否覆盖了主要竞品（≥2个）？

### Step 2：竞品对标分析
- 将本产品和竞品在9大标签模块上做**维度对比**
- **重点维度**：
  - 使用阻碍（痛点解决能力）
  - 预期符合度（满意度）
  - 性价比感受（价格竞争力）
  - 改进建议（产品方向）
- **输出**：竞品对标矩阵（本产品 vs 竞品A vs 竞品B）

### Step 3：JTBD框架分析（竞品视角）
- 分析用户选择竞品时想要完成的"任务"是什么
- 对比本产品和竞品在满足JTBD上的表现
- **输出**：JTBD竞品对比表

### Step 4：十三种需求分析（需求覆盖对比）
- 将本产品和竞品的需求覆盖做对比
- 找出竞品覆盖但本产品未覆盖的需求（机会点）
- 找出本产品覆盖但竞品未覆盖的需求（差异化点）
- **输出**：需求覆盖对比矩阵

### Step 5：极端用户识别（竞品转换用户）
- **重点**：识别从竞品转换到本产品的用户
- 分析转换动机（为什么换？）
- 分析转换后的满意度（换了之后满意吗？）
- **输出**：竞品转换用户洞察

### Step 6：整合输出成品报告
按以下结构输出（详见「输出报告结构」章节）

## AI执行注意事项

1. **必须有多产品数据**：竞品分析必须有竞品数据，只有本产品数据无法做对标
2. **直接引用对比**：每条结论必须有本产品 + 竞品的评论引用
3. **避免主观评价**：差异化优势/劣势必须有用户评论证据，不可AI主观判断
4. **转换用户是金矿**：从竞品转换过来的用户的评论，直接揭示竞品弱点
5. **性价比需量化**：尽可能用具体价格、性价比评分来支撑结论

## 质量自检清单

- [ ] 是否有至少2个竞品的数据？
- [ ] 对标矩阵是否覆盖了核心维度（≥5个）？
- [ ] 差异化优势是否有用户评论引用？
- [ ] 竞品机会是否有竞品评论引用？
- [ ] 转换用户洞察是否有具体评论证据？
