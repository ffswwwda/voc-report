# VOC分析协议 · 用户洞察目的

> 本文件为AI分析执行协议。AI读取本协议 + 原始VOC数据，直接输出成品分析报告。

---

## 元数据

| 字段 | 内容 |
|------|------|
| 分析目的 | 用户洞察 |
| 核心问题 | 用户为什么买/不买？用户是谁？用户真正在意什么？ |
| 适用场景 | 用户画像构建、用户旅程地图、满意度分析、新市场进入 |
| 预计分析时长 | 45-60分钟 |

---

## 输入数据规范

### 主数据源
- **Amazon评价（全量1-5星）**：完整用户反馈 spectrum
- **Reddit讨论（全量）**：深度用户表达、社群讨论

### 辅助数据源
- **社媒反馈**：用户自发分享、使用场景描述
- **客服对话记录**：用户直接反馈的问题

### 数据预处理要求
1. **保留全量数据**（1-5星全部保留，不可过滤）
2. 按星级分层：1-2星（痛点）、3星（中立）、4-5星（满意点）
3. 标记评论长度（长评论通常包含更深层的洞察）

---

## 分析步骤（严格按顺序执行）

### Step 1：标签打标
对每条原始评论，按9大标签模块打标：

| 优先级 | 标签模块 | 打标重点 |
|--------|-----------|-----------|
| ⭐⭐⭐⭐⭐ | ⑨用户画像/人群 | **核心**：构建完整用户画像 |
| ⭐⭐⭐⭐⭐ | ①使用场景 | 还原用户真实使用场景 |
| ⭐⭐⭐⭐ | ②购买动机 | 理解用户深层驱动力 |
| ⭐⭐⭐⭐ | ③预期符合度 | 分析满意/不满意的根源 |
| ⭐⭐⭐⭐ | ⑦用户忠诚度 | 识别品牌关系类型 |
| ⭐⭐⭐ | ④性价比感受 | 理解用户价值感知 |
| ⭐⭐⭐ | ⑤使用阻碍 | 用户遇到的真实问题 |
| ⭐⭐ | ⑥克服方式 | 用户如何应对问题 |
| ⭐⭐ | ⑧产品改进建议 | 用户的直接建议 |

- **输出**：打标后的结构化数据（含用户ID、星级、标签）
- **质量检查**：用户画像模块覆盖率需≥90%

### Step 2：用户画像聚合分析
- 从标签数据中提取用户特征维度：
  - **人口统计学特征**：年龄、性别、职业（从评论文本推断）
  - **行为特征**：使用频率、使用场景、购买渠道
  - **心理特征**：购买动机、价值观、需求层次
- **输出**：用户画像报告（3-5类核心用户群体）

### Step 3：JTBD框架分析（用户任务洞察）
- 分解用户的三层次任务：
  - **功能性任务**：用户要完成的具体操作
  - **情感性任务**：用户使用时的感受目标
  - **社会性任务**：用户想向他人传达的形象
- **输出**：JTBD用户任务地图

### Step 4：十三种需求分析（需求层次洞察）
- 将用户需求映射到十三种需求模型
- **重点**：识别用户需求的主导层次（生存/安全/社交/尊重/自我实现）
- **输出**：用户需求层次分布图

### Step 5：研究员职责视角分析
- 用四象限框架分析用户需求类型：
  - **表达出的需求** vs **真实需求**
  - **战略性需求** vs **临时性需求**
- **输出**：用户需求类型洞察

### Step 6：整合输出成品报告
按以下结构输出（详见「输出报告结构」章节）

## AI执行注意事项

1. **全量数据保留**：不可过滤任何星级评论，1星和5星同等重要
2. **用户画像需有证据支撑**：每个画像特征必须有原文引用
3. **区分相关性和因果性**：用户特征和相关行为不一定有因果关系
4. **情感曲线需有数据支撑**：用户旅程的每个阶段情感需有评论证据
5. **用户细分需可执行**：细分结果必须能指导具体行动（产品/营销）
