# 需求四象限分析技能（重要度-满意度矩阵）

## 目的

将VOC数据中分散的用户需求，通过**重要度-满意度矩阵**系统化划分为4个象限，明确产品改进优先级，避免资源浪费在低价值需求上。

---

## 一、得分计算方式

### 1.1 重要度得分（Importance Score）

从VOC数据中提取每个需求点（tag/item）的重要度得分：

```
重要度得分 = 提及频次权重 × 情绪强度权重 × 星级关联权重
```

| 数据来源 | 计算方式 | 说明 |
|---------|---------|------|
| 提及频次 | `count(tag)` / `总评论数` | 越多人提及，越重要 |
| 情绪强度 | 正面情绪均值 − 负面情绪均值（−1~1） | 情绪越强烈，越重要 |
| 星级关联 | 提及该需求时的平均星级 | 低星级+提及该需求 = 重要度高 |
| 加权公式 | `频次×0.4 + 情绪×0.3 + 星级×0.3` | 可调整权重 |

**输出**：每个需求点的 `importance_score`（范围 0~10）

### 1.2 满意度得分（Satisfaction Score）

```
满意度得分 = 正面评价占比 − 负面评价占比 + 平均星级（标准化到0~10）
```

| 数据来源 | 计算方式 | 说明 |
|---------|---------|------|
| 正面评价占比 | `正面提及次数` / `总提及次数` | 用户对该需求的满意比例 |
| 负面评价占比 | `负面提及次数` / `总提及次数` | 用户对该需求的不满比例 |
| 平均星级 | （平均星级 − 1）× 2.5 | 标准化到0~10 |
| 加权公式 | `正面占比×5 + (1−负面占比)×5` | 简化版 |

**输出**：每个需求点的 `satisfaction_score`（范围 0~10）

---

## 二、象限划分规则

以**中位数**（Median）为切割线，将需求点划分到4个象限：

```
切割线 = MEDIAN(所有需求点的重要度得分)   ← 纵轴
切割线 = MEDIAN(所有需求点的满意度得分)   ← 横轴
```

| 象限 | 重要度 | 满意度 | 对应含义 |
|-------|--------|--------|---------|
| 核心优势区（左上） | ≥ 中位数 | ≥ 中位数 | 用户最在意，产品也做得好 |
| 重点改进区（右上） | ≥ 中位数 | < 中位数 | 用户最在意，但产品做得差 ⚡ |
| 过度投入区（左下） | < 中位数 | ≥ 中位数 | 用户不太在意，产品却投入很多 |
| 低优先级区（右下） | < 中位数 | < 中位数 | 用户不在意，产品也做得差 |

> **替代切割方式**：
> - 按行业基准（竞品对标）划分
> - 按产品战略目标划分（非中位数）
> - 按Kano模型分类划分

---

## 三、四个象限的优先级、关系与用途

### 优先级排序

```
重点改进区 ⚡（最高优先级）
    ↓ 资源优先投入
核心优势区 ✅（保持投入）
    ↓ 维持现有水平
低优先级区 ⏳（延后处理）
    ↓ 最后考虑
过度投入区 ⬇（减少投入）
    ↓ 释放资源
```

### 各象限的战略用途

| 象限 | 战略用途 | 具体行动 |
|-------|---------|---------|
| **重点改进区** | 产品改进最高优先级 | 分配核心研发资源；设定改进KPI；追踪满意度变化 |
| **核心优势区** | 营销卖点 + 保持优势 | 写入产品详情页；用于广告素材；保持现有投入水平 |
| **过度投入区** | 减少投入 + 释放资源 | 降低研发投入；维持基本水平；将资源转移至重点改进区 |
| **低优先级区** | 延后处理 + 监控 | 资源充裕时再改进；定期重新评估；关注竞品动态 |

### 象限之间的关系

```
过度投入区 ──资源释放──▶ 重点改进区
   ↑                                      ↓
   │                              满意度提升后
   │                              移动到核心优势区
   │                                      ↓
低优先级区 ──监控变化──▶ 核心优势区（如果需求重要性上升）
```

---

## 四、每个象限内部的排序规则

每个象限内部的需求点，按以下规则排序后输出：

### 排序公式

```
重点改进区排序 = 重要度得分 − 满意度得分（差值越大，优先级越高）
核心优势区排序 = 重要度得分 + 满意度得分（总分越高，优势越明显）
过度投入区排序 = 满意度得分 − 重要度得分（差值越大，浪费越严重）
低优先级区排序 = 重要度得分（按重要性从低到高，最后处理）
```

### 输出格式示例

```markdown
## 重点改进区 ⚡（最高优先级）

| 排名 | 需求点 | 重要度 | 满意度 | 差值 | 具体痛点 |
|------|--------|--------|--------|------|---------|
| 1 | 清洗便捷性 | 8.9 | 4.2 | 4.7 | "清洗很麻烦，用完还要洗" |
| 2 | 噪音控制 | 8.3 | 3.8 | 4.5 | "怕被室友听到声音" |
| 3 | 材质真实感 | 7.8 | 4.5 | 3.3 | "希望材质更接近真实皮肤" |

## 核心优势区 ✅（保持优势）

| 排名 | 需求点 | 重要度 | 满意度 | 总分 | 可作为卖点 |
|------|--------|--------|--------|------|---------|
| 1 | 材质安全 | 8.7 | 8.2 | 16.9 | 食品级材质，无毒认证 |
| 2 | 物流速度 | 7.9 | 8.5 | 16.4 | 隐私发货，当日达 |
| 3 | 包装隐私性 | 7.5 | 7.8 | 15.3 | 无标识保密包装 |
```

## 五、完整分析流程

```
Step 1：数据输入
  └─ VOC原始数据（评论文本、星级评分、情绪标签、提及频次）

Step 2：需求点提取
  └─ 从VOC数据中提取所有需求点（可复用9大标签模块的分类）

Step 3：计算重要度和满意度得分
  └─ 每个需求点输出：importance_score, satisfaction_score

Step 4：象限划分
  └─ 以中位数为切割线，划分4个象限

Step 5：象限内排序
  └─ 按各象限的排序规则，对每个需求点排序

Step 6：输出报告
  └─ 四象限矩阵可视化 + 各象限需求列表 + 行动建议
```
