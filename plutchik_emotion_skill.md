# Plutchik 情绪罗盘 · 用户情绪分析技能

## 技能定位

本技能基于**Robert Plutchik的情绪轮（Wheel of Emotions）理论**，提供系统化的用户情绪识别与情感驱动分析方法。从VOC数据（用户评论、社媒讨论、客服对话）中提取情绪信号，量化用户的情感反应模式，并将情绪映射到具体行为，辅助产品改进、差评诊断与营销优化。

## 理论基础：Plutchik 情绪轮

### 核心观点
- 人类有 **8 种基本情绪**，排列在一个色轮（wheel）结构上
- 每种基本情绪有 **3 级强度**（低 / 中 / 高），越靠圆心强度越低
- **相邻情绪可混合**，产生复合（次级）情绪，如 喜+信=爱
- 相对位置的情绪互为"对立"（如 喜 ↔ 哀，怒 ↔ 恐）

### 8 种基本情绪与强度

| 基本情绪 | 中文 | 低强度 | 中强度 | 高强度 |
|---------|------|--------|--------|--------|
| Joy | 喜 | 平静 Serenity | 高兴 Joy | 狂喜 Ecstasy |
| Trust | 信 | 接纳 Acceptance | 信任 Trust | 崇拜 Admiration |
| Fear | 恐 | 担忧 Apprehension | 害怕 Fear | 恐怖 Terror |
| Surprise | 惊 | 分心 Distraction | 惊讶 Surprise | 惊愕 Amazement |
| Sadness | 哀 | 沮丧 Pensiveness | 失望 Sadness | 悲痛 Grief |
| Disgust | 厌 | 厌腻 Boredom | 反感 Disgust | 憎恶 Loathing |
| Anger | 怒 | 烦躁 Annoyance | 生气 Anger | 暴怒 Rage |
| Anticipation | 盼 | 兴趣 Interest | 期待 Anticipation | 警惕 Vigilance |

### 常见复合情绪（相邻混合）

| 混合 | 复合情绪 | 含义 |
|------|---------|------|
| 喜 + 信 | 爱 Love | 对品牌/产品的喜爱 |
| 信 + 恐 | 服从 Submission | 被动接受、无力反抗 |
| 恐 + 惊 | 敬畏 Awe | 又怕又惊奇 |
| 惊 + 哀 | 否认 Disapproval | 失望夹杂意外 |
| 哀 + 厌 | 愧 Remorse | 后悔购买 |
| 厌 + 怒 | 蔑视 Contempt | 强烈负面、鄙视 |
| 怒 + 盼 | 攻击性 Aggressiveness | 主动投诉维权 |
| 盼 + 喜 | 乐观 Optimism | 期待且满意 |

## 核心能力

1. **情绪分类打标**：将用户评论标注为 8 种基本情绪之一
2. **强度判定**：3 级强度（低/中/高）量化情绪强烈程度
3. **复合情绪识别**：检测相邻情绪混合产生的次级情绪
4. **情绪→行为映射**：将情绪类型映射到用户行为（复购/推荐/流失/投诉）
5. **可视化报告**：生成情绪分布玫瑰图与洞察卡片

## 使用流程

### 步骤1：理解用户输入

用户可能以以下方式提问：
- "帮我分析一下飞机杯评论的情绪分布"
- "为什么这条差评情绪这么负面？"
- "用户对产品的情感倾向是正面还是负面？"
- "从情绪角度给我一些营销文案建议"

**关键信息提取**：
- 产品类型/领域
- 数据源类型（电商评价/Reddit/社媒/客服对话）
- 分析目标（差评诊断/好评提炼/品牌监测/文案优化）

### 步骤2：情绪打标（8 基本情绪）

对每条用户评论，判定其主要情绪：

| 情绪 | 典型触发词（中英文） |
|------|---------------------|
| 😊 喜 Joy | 满意、喜欢、完美、物超所值、best、love it、amazing |
| 🤝 信 Trust | 信赖、安心、包装隐蔽、正品、值得买、reliable |
| 😨 恐 Fear | 怕被发现、担心隐私、怕质量差、worried、scary |
| 😲 惊 Surprise | 没想到、居然、意外、surprised、didn't expect |
| 😢 哀 Sadness | 失望、落差、没想象好、regret、let down |
| 🤢 厌 Disgust | 异味、恶心、难清洗、gross、disgusting、cheap feel |
| 😡 怒 Anger | 垃圾、浪费钱、差评、退货、angry、worst、terrible |
| 🤔 盼 Anticipation | 希望、期待、等下一代、hope、wish、looking forward |

### 步骤3：强度判定（3 级）

```
低（★☆☆）：隐含、微弱
  例："还可以吧" / "一般般"
中（★★☆）：明显、可感知
  例："挺满意的" / "有点失望"
高（★★★）：强烈、爆发式
  例："太棒了完美！" / "垃圾再也不买！"
```

### 步骤4：复合情绪识别

检测同一条评论中出现的"相邻情绪混合"：
```
"包装很隐蔽我很放心（信），但材质味道太重（厌）"
→ 信 + 厌 = 矛盾心理（安心但排斥）
→ 洞察：隐私做得好，但材质气味是硬伤，优先级高于包装
```

### 步骤5：情绪→行为映射

| 情绪组合 | 行为指向 | 业务含义 |
|---------|---------|---------|
| 喜（高）+ 信（中） | 复购 / 主动推荐 | ✅ 核心正向驱动 |
| 盼（低）+ 喜（中） | 观望等待促销 | 💡 降价/活动可转化 |
| 哀（中） | 沉默流失 | ⚠️ 不写差评但不再买 |
| 怒（高）+ 厌（高） | 差评 / 投诉维权 | ❌ 致命缺陷暴露 |
| 恐（中） | 隐私需求强化 | ⚠️ 需强调 discreet 设计 |

### 步骤6：生成可视化HTML报告

1. **情绪分布统计表**：8 情绪 ×（提及量 / 占比 / 平均强度 / 典型文本 / 行为指向）
2. **情绪玫瑰图**：用饼图/玫瑰图展示 8 情绪占比
3. **情绪→行动映射卡片**：正向驱动 / 负向驱动 / 中性预警 / 未来需求 四格洞察
4. **与标签体系映射**：情绪标签作为 Module-3（情感倾向）的二级维度
5. **行动建议**：按情绪优先级排序的改进项

## 与标签体系的映射关系

Plutchik 情绪标签可作为 **标签体系 Module-3（情感倾向）** 的细化维度：

| Module-3 粗粒度 | Plutchik 二级细化 |
|---------------|-----------------|
| 正面情感 | 喜 / 信 / 惊(正向) |
| 中性情感 | 盼 / 低强度哀 |
| 负面情感 | 怒 / 厌 / 恐 / 高强度哀 |

**用法**：先打 Module-3 粗粒度标签（正面/负面/中性），再对需要深挖的评论用 8 基本情绪做二级标注，实现从粗到细的情感分析。

## 数据源适配指南

| 数据源 | 情绪信号清晰度 | 强度可判性 | 推荐用法 |
|--------|--------------|-----------|---------|
| **电商评价** | ✅ 评分+文字 | ✅ 高（极端评分明确） | **首选**：差评/好评诊断 |
| **Reddit/社媒** | ✅ 长文本情感丰富 | ⚠️ 中（需语境） | 趋势/复合情绪挖掘 |
| **客服对话** | ✅ 实时情绪 | ✅ 高（怒/恐明显） | 危机预警 |
| **访谈记录** | ✅ 深度情感 | ⚠️ 中 | 补充洞察 |

## 输出质量标准

**好的情绪分析报告特征**：
- ✅ 理论支撑（基于 Plutchik 8 情绪模型）
- ✅ 量化分布（每个情绪有提及量与占比）
- ✅ 行为映射（情绪不是终点，要落到复购/流失/投诉）
- ✅ 优先级明确（哪些情绪对应 P0 改进）

**避免的输出**：
- ❌ 只说"用户情绪负面"而无具体情绪类型
- ❌ 忽略强度（"有点失望"和"垃圾"严重程度完全不同）
- ❌ 情绪与行为脱节（识别了情绪但没说意味着什么业务动作）
